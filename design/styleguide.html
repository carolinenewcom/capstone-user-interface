Color Theme

My application incorporates the below color scheme in its interface.

I chose variations of blue to be my color scheme. Blue is thought to be a color that represents reliability. People tend to associate the color blue with thoughts of order and stability. Those are all things that my application seeks to provide. The aim is for users to feel as though the content they are viewing is well organized, trust-worthy, and reliable. I am hoping to indirectly communicate this through the colors used throughout the application.

Buttons

The application will feature buttons that contain the below color and shape:

The website will have different types of buttons. A 'Submit' button will be displayed for when a user needs to submit a form or a search input. An 'Add Child' button will be on display for when users need to add a new child to their caseload within the application. An 'Add Session' button will be on display for when a user would like to add a new session to their schedule within the application. A 'Cancel' button will be on display for when a user would like to exit the form they are in. The application will also include radio buttons for users to select the session attendance for each week.

Messages

For various completed tasks, application notifications will be presented to the user based. The following contextual messages will be used.

An error message will be displayed in red as it is the most commonly used color for error messages. The user will quickly see that an error has occured such as entering an invalid date. The success message will use a blue color as found within the color scheme pallet to continue the color uniformity within the website. Users will be able to see if the information they have inputted has been submitted successfully.
